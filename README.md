# Overview
The kube-sec-board deployment monitors the kubernetes cluster for updates in configauditreports and vulnerabilityreports. When one of these reports is regenerated by the Trivy operator deployment it is processed. For each Trivy issue a task is created in kanboard. Two kanboard projects are created for each kubernetes namespace. one for the config audit reports, and one for the vulnerabilities. When a change is detected for eaxmple when part of an issue is solved, the kanboard issue is updated or closed. When an issue is reappearing the kanboard task is reopened.

# Dependencies
The Trivy operator and Kanboard deployments must be active in the environment.

# Environment variables
The following enviroment variables are supported:
| Variable | Description |
| -------- | -------- |
| KANBOARD_URL | The Kanboard url for API access, f.e. http://kanboard.example.com/jsonrpc.php |
| KANBOARD_USER | The kanboard API user |
| KANBOARD_APIKEY | The Kanboard API key |
| NAMESPACES_EXCLUDED | (optional) The comma separated list of namespaces to exclude, defaults to none |
| NAMESPACES_INCLUDED | (optional) The comma separated list of namespaces to include, defaults to all |
| CAR_SEVERITIES | (optional) The comma separated list of configauditreport severities to process, defaults to all. Possible values are: UNKNOWN,LOW,MEDIUM,HIGH,CRITICAL |
| VULN_SEVERITIES | (optional) The comma separated list of vulnerabilityreport severities to process, defaults to all. Possible values are: UNKNOWN,LOW,MEDIUM,HIGH,CRITICAL |
